# 四期实施路线图

> SkyHub CSM AI 系统从 0 到 1 再到规模化的完整落地计划，按季度分四大阶段推进。

---

## 总览

| 阶段 | 时间 | 主题 | 核心目标 |
|------|------|------|----------|
| Phase 1 | M1-4 | 基座内核 | Obsidian KB + 4 个高频 Skill + 5-8 个 Champion |
| Phase 2 | M4-8 | 流程深化 | Playbook 引擎 + 3 个新 Skill + 全员培训 + 采纳率 >= 50% |
| Phase 3 | M8-14 | 数据驱动 | 健康分校准 + NRR/GRR 看板 + VoC 闭环 + 行业垂直方案 |
| Phase 4 | M14+ | 规模拓展 | 多语言 + Skill 市场 + L4/L5 升级 + 最佳实践输出 |

---

## Phase 1: 基座内核 (Month 1-4)

**Theme:** 建基座，跑通核心闭环，让 CSM "用起来"

### 月度交付物

#### M1 -- 数据基座与基础设施

- Obsidian Vault 初始化：目录结构、模板、frontmatter schema 定稿
- 客户数据批量导入：覆盖率 >= 80% 的存量客户
- MCP Hub 基础搭建：CRM（HubSpot/Salesforce）读写对接
- 权限体系设计：按角色（CSM / Manager / Ops）分层访问
- 环境部署：开发 + Staging + Production 三套环境就绪

#### M2 -- 首批 Skill 上线

- **KB-QA** 上线：知识库问答命中率 >= 85%；覆盖产品 FAQ、流程 SOP、政策文档
- **Empathy-Writer** 上线：话术满意度 >= 4/5；支持邮件、IM、工单三种场景
- Obsidian 模板库 v1：客户档案、会议纪要、交接文档三类模板
- 内部 Beta 测试：3-5 名种子用户试用并提交反馈

#### M3 -- 核心 Skill 补全

- **Health-Dashboard** 上线：每日自动更新，5 维度评分，Dataview 基础仪表盘
- **Sales-Handoff** 上线：交接文档完整度 >= 90%，交接时间较手工减少 >= 50%
- Dataview 基础仪表盘：客户概览、健康分布、到期日历
- 告警规则 v1：健康分降级自动通知 CSM

#### M4 -- Champion 验证

- Champion 用户培训完成：5-8 人，日均使用 >= 5 次
- 用户反馈收集与迭代：至少完成一轮 Skill 优化
- Phase 1 评估报告：含采纳数据、ROI 初估、改进清单
- Phase 2 启动准备：范围确认、资源分配

### 成功标准

| 指标 | 目标值 | 测量方式 |
|------|--------|----------|
| Champion 日均调用 | >= 5 次 | MCP 调用日志 |
| KB 准确率 | >= 85% | 人工抽检 + 用户反馈 |
| 话术满意度 | >= 4.0/5 | CSM 评分 |
| 数据导入率 | >= 80% | Vault 覆盖率统计 |
| 交接时间缩短 | >= 50% | 前后对比 |

## Phase 2: 流程深化 (Month 4-8)

**Theme:** 从 "工具" 到 "流程引擎"，让 AI 嵌入日常工作流

### 月度交付物

#### M5 -- Playbook 引擎

- Playbook 引擎 v1 上线：支持顺序、条件、并行三种编排
- 首批 3 条 Playbook：Onboarding-90 天、风险干预、续约倒计时
- **TTV-Tracker** 上线：自动追踪首次价值实现里程碑

#### M6 -- 全员推广

- 全 CSM 团队培训：分批次 workshop，覆盖率 100%
- **Adoption-Analyzer** 上线：功能使用热力图、登录趋势、活跃度评分
- 新 Playbook：扩展销售、QBR 准备

#### M7 -- 流程优化

- **Voice-of-Customer** 上线：多渠道情感分析（工单 + 邮件 + NPS）
- Playbook 效果追踪：完成率、耗时、结果关联
- CSM 个人仪表盘：任务清单 + 建议优先级

#### M8 -- 阶段评估

- 全员采纳率评估，目标 >= 50%
- Playbook 完成率 >= 70%
- QBR 准备时间缩短 >= 60%
- Phase 2 评估报告，Phase 3 启动

### 成功标准

| 指标 | 目标值 | 测量方式 |
|------|--------|----------|
| 全员日均调用 | >= 3 次 | MCP 调用日志 |
| 采纳率 | >= 50% | 周活跃用户/总 CSM |
| Playbook 完成率 | >= 70% | 引擎追踪 |
| QBR 准备时间缩短 | >= 60% | 前后对比 |

## Phase 3: 数据驱动 (Month 8-14)

**Theme:** 从 "流程驱动" 到 "数据驱动"，用数据指导决策

### 月度交付物

#### M9-M10 -- 预测模型

- **Churn-Predict** 上线：基于历史数据的流失预测，准确率 >= 60%
- 健康分模型校准：对照实际续约/流失结果，持续调参
- NRR/GRR 实时看板上线

#### M11-M12 -- 闭环运营

- VoC 闭环机制：从洞察 -> 行动 -> 验证的完整循环，闭环率 >= 80%
- **Renewal-War-Room** 上线：续约作战室，整合风险 + 竞品 + 价值数据
- **Competitor-Radar** 上线：竞品动态追踪与防御话术

#### M13-M14 -- 行业方案

- 行业垂直方案 v1：至少 2 个行业（如 SaaS + 制造业）的定制 Playbook
- **Value-Proof-Generator** 上线：自动生成 ROI 报告、价值证明文档
- **Executive-Briefing** 上线：高管汇报自动化
- Phase 3 评估报告

### 成功标准

| 指标 | 目标值 | 测量方式 |
|------|--------|----------|
| 流失预测准确率 | >= 60% | 预测 vs 实际 |
| 预测遗漏率 | <= 15% | 未被预警的流失 |
| VoC 闭环率 | >= 80% | 洞察 -> 行动追踪 |
| NRR 提升 | +3-5pp | 前后对比 |

## Phase 4: 规模拓展 (Month 14+)

**Theme:** 全球可复制模型，从项目到产品

### 核心举措

#### 多语言与全球化

- 英文 / 日文 / 繁中 Skill 版本上线
- 本地化知识库：各区域产品差异、合规要求、文化适配
- 多时区 Playbook 触发与通知

#### Skill 市场

三级体系：

| 层级 | 说明 | 审核 |
|------|------|------|
| Official | 官方出品，核心功能 | 内部团队全量测试 |
| Certified | 合作伙伴开发，官方认证 | 功能 + 安全审核 |
| Community | 社区贡献，自担风险 | 基础格式校验 |

- 开发者文档与 SDK
- Skill 评分与评价系统
- 收入分成模型（如适用）

#### L4/L5 自动化升级

- L4 候选场景评估：邮件自动回复、续约提醒自动发送、数据报表自动分发
- L5 候选场景评估：简单续约自动处理、低风险工单自动关闭
- 人工监督与回退机制

#### 最佳实践输出

- 行业白皮书：CSM AI 最佳实践
- 客户案例集（获得客户授权）
- 内部培训认证体系

## 变革管理

### 四阶段培训

| 阶段 | 时间 | 对象 | 方式 | 目标 |
|------|------|------|------|------|
| 种子用户 | M1-4 | Champion (5-8人) | 1:1 深度辅导 | 日均使用 >= 5 次 |
| 全员推广 | M5-8 | 全部 CSM | 分批 Workshop | 采纳率 >= 50% |
| 持续精进 | M8-14 | 全团队 | 月度分享会 + 新 Skill 培训 | 高级功能使用率 >= 30% |
| 自我进化 | M14+ | 全组织 | 内部认证 + 社区共创 | CSM 可自行配置 Playbook |

### 五级信任阶梯

CSM 对 AI 系统的信任度逐步提升，驱动更高层级的人机协作：

| 等级 | 名称 | 采纳率区间 | 行为特征 | 推进策略 |
|------|------|-----------|----------|----------|
| L1 | Observer | < 20% | 观望，偶尔查看 AI 输出 | 展示价值案例，消除顾虑 |
| L2 | Referrer | 20-30% | 将 AI 输出作为参考之一 | 提供对比数据，证明准确性 |
| L3 | Collaborator | 40-55% | 日常工作中主动使用 AI | 嵌入工作流，减少切换成本 |
| L4 | Delegator | 55-70% | 信任 AI 处理常规任务 | 开放更多自动化场景 |
| L5 | Co-creator | 65-80% | 与 AI 共同创新工作方式 | 鼓励自定义 Playbook 与 Skill |

### Champion 培养

#### 选拔标准

- 对新工具持开放态度，愿意尝试
- 在团队中有一定影响力
- 管理客户数 >= 10，覆盖典型场景
- 每周可投入 >= 2 小时参与测试与反馈

#### 六步培养路径

```
选拔 → 深度培训 → 实战验证 → 内部布道 → 持续进化 → 认证
```

1. **选拔 (M1 W1-W2):** 按标准选出 5-8 名候选人，签署参与协议
2. **深度培训 (M1 W3-W4):** 1:1 培训全部已上线 Skill，熟悉 Obsidian Vault 操作
3. **实战验证 (M2-M3):** 在真实客户场景中使用，每周反馈会，快速迭代
4. **内部布道 (M3-M4):** Champion 在团队会议中分享经验，制作使用 Tips
5. **持续进化 (M4+):** 参与新 Skill Beta 测试，贡献 Playbook 模板
6. **认证 (M8+):** 通过考核获得 "SkyHub AI Champion" 认证，纳入绩效加分
